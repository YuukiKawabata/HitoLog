# HitoLog Privacy Policy Draft

HitoLog collects the minimum information needed to provide account, posting, safety, and notification features.

## Information Collected
- Account identifiers from Sign in with Apple and Firebase Auth.
- Profile information entered by the user, including display name, handle, and bio.
- Posts, comments, likes, blocks, mutes, and reports created in the app.
- Typing-derived integrity signals for posts, including input duration, edit count, delete count, and suspicious bulk input count.
- Firebase Cloud Messaging tokens when the user enables notifications.
- Feedback submitted from Settings, including optional contact email when the user requests a reply.
- Usage analytics events such as screen views and button actions when analytics collection is enabled. Post and feedback body text is not included in analytics events.

## Use of Information
Information is used to operate the social feed, display user profiles, maintain safety controls, calculate Human Check labels, process reports, improve product quality from feedback and usage analytics, and deliver notifications for comments and likes.

## Notifications
Notification tokens are used only to send HitoLog notifications. Users can disable notifications in the app or in iOS Settings.

## Data Sharing
HitoLog uses Firebase services for authentication, database storage, App Check, Cloud Messaging, and Cloud Functions. HitoLog uses PostHog for product analytics when enabled. Data is not sold.

## Account Deletion
Users can request account deletion from Settings. Deletion disables notification delivery, removes the Firebase Auth user, marks the user's profile, posts, and comments as deleted, removes likes and safety preference records created by the user, deletes saved notification tokens, and returns the app to the initial state. Reports may be retained as moderation records when required for safety review.

## Contact
Support email and public policy URL must be filled in before App Store Connect submission.
